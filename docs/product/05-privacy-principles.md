# Privacy Principles

Privacy is not a secondary feature of Pensieve. It is central to the product.

## Local First

Raw development activity should remain on the user's machine by default.

## User Controlled

The developer decides:

* which data sources Pensieve can access;
* which projects are included;
* which insights are retained;
* what information is exported;
* and whether anything is shared.

## No Manager Dashboard

Pensieve is not designed as a workforce surveillance system.

There is no automatic management feed, productivity score, or centralized employee-ranking interface.

## Transparent Processing

The user should be able to inspect:

* which source activity produced an insight;
* why Pensieve classified it as meaningful;
* what data was used;
* and how the output was generated.

## Editable Memory

Pensieve's summaries are interpretations, not objective truth.

The developer must be able to edit, reject, merge, or delete any generated insight.
