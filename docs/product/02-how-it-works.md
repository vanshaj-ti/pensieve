# How Pensieve Works

## 1. Local Data Ingestion

Pensieve integrates at the local machine level and reads data generated by the developer's existing tools.

Initial sources may include:

* Cursor chat histories stored in local application databases;
* Claude Code conversation logs;
* terminal session histories;
* local Git metadata;
* selected IDE events;
* developer-authored notes and markdown files.

The initial product does not require organization-wide installation, cloud-level source control access, or a management-controlled analytics system.

Pensieve is designed to work for the individual developer first.

## 2. Semantic Extraction

A specialized local analysis agent processes recent work sessions and identifies high-value cognitive signals.

The system does not attempt to summarize every prompt or command. Instead, it searches for moments such as:

* architectural decisions;
* corrections to AI-generated approaches;
* system constraint identification;
* risk and edge-case detection;
* design trade-offs;
* debugging breakthroughs;
* product and workflow insights;
* repeated tooling friction;
* abandoned but promising ideas.

Pensieve removes boilerplate, repetitive code generation, and low-information interactions so that the final record focuses on the developer's meaningful contribution.

## 3. Decision and Insight Reconstruction

Pensieve groups individual interactions into coherent work episodes.

For example, a long conversation involving database schemas, API changes, failed implementations, and test failures may be reconstructed as:

> Identified that the proposed background-job design could create duplicate processing under retry conditions. Reframed the implementation around idempotent operations and introduced a unique execution key before code generation continued.

This transforms fragmented chat activity into a concise account of the developer's reasoning and impact.

## 4. Private Reflection Workspace

Pensieve provides a personal interface where developers can review, edit, reject, or expand the insights generated by the system.

The developer remains the final authority over what is retained.

The workspace may include:

* a chronological decision timeline;
* a searchable archive of insights;
* recurring friction patterns;
* saved ideas and unexplored opportunities;
* weekly summaries;
* personal development themes;
* evidence prepared for one-on-ones and performance reviews.

## 5. The Weekly Pensieve Brief

Every week, Pensieve generates a private briefing that summarizes the developer's most meaningful work.

The briefing may include:

* key technical decisions;
* important corrections and interventions;
* architectural or product insights;
* recurring sources of friction;
* high-potential ideas;
* unresolved questions;
* examples of leverage created through AI;
* examples where AI caused unnecessary rework.

The output can be delivered as a markdown document, local dashboard, or exportable review summary.
